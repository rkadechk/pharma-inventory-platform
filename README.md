## 📋 Project Overview

### Problem Statement
Healthcare facilities waste **$4.5B annually** on medications due to:
- Medication expiration (shelf-life constraints)
- Demand uncertainty (seasonal spikes, outbreaks)
- Poor coordination between facilities (excess in one, shortage in another)
- Reactive systems relying on manual processes

### Solution
A **full-stack AI-powered platform** that brings together:
- Three specialized AI agents coordinated by an orchestrator
- A RESTful FastAPI backend exposing all functionality via API
- An interactive Streamlit dashboard for real-time monitoring
- A comprehensive 7-stage data validation pipeline
- Synthetic healthcare data (~150,000+ records across 8 CSV files)

### Impact
- ✅ Reduces medication waste by 30–50% through proactive expiration management
- ✅ Prevents drug shortages through AI-driven inter-facility transfer matching
- ✅ Forecasts demand 30 days ahead with Prophet + ARIMA models
- ✅ Supports regulatory compliance via full audit trail and validation pipeline

## 🤖 AI Agent System

### Three Specialized Agents

**1. DemandForecastingAgent** (`agents/demand_agent.py`)
- Predicts medication demand **30 days ahead**
- Models: Meta's **Prophet** (primary), **ARIMA** (fallback)
- Anomaly detection: **Z-score** (flag at >2σ MEDIUM, >3σ HIGH) + spike/dip pattern detection
- Incorporates external signals: weather, disease outbreaks, seasonal trends

**2. InventoryOptimizationAgent** (`agents/inventory_agent.py`)
- Scans 3,000 inventory batches across 5 facilities
- Classifies expiration risk: CRITICAL / HIGH / MEDIUM / LOW
- Detects stockout risk and overstock conditions
- Generates transfer recommendations ranked by urgency

**3. SupplyChainCoordinationAgent** (`agents/supply_chain_agent.py`)
- Decides: reorder from supplier vs. inter-facility transfer
- Multi-criteria optimization: distance, cost, urgency, capacity
- Balances inventory levels across the facility network

### Orchestrator (`agents/orchestrator.py`)
Coordinates all three agents in a sequential pipeline:
1. Run demand forecasts → identify upcoming surges
2. Analyze inventory risk → flag expiry and stockout threats
3. Resolve with supply chain decisions → reorder or transfer
4. Synthesize into a unified action plan with prioritized recommendations

### CrewAI Integration (`agents/crewai_pharmacy.py`)
LLM-enhanced agents powered by **CrewAI + LangChain + GPT-4o** provide natural-language reasoning and explanation for recommendations. Agent memory is enabled for context retention across interactions.

## 🚀 Getting Started

### Prerequisites
- Python 3.8+
- OpenAI API key (for CrewAI / GPT-4o agents)
- Anthropic API key (optional, for Claude)

### Setup
```bash
# Clone the repo
git clone https://github.com/rkadechk/pharma-inventory-platform.git
cd pharma-inventory-platform

# Create and activate virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.template .env
# Edit .env with your API keys
```

### Run the Platform

**FastAPI backend:**
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```
API docs available at `http://localhost:8000/docs`

**Streamlit dashboard:**
```bash
streamlit run dashboard.py
```
Dashboard available at `http://localhost:8501`

### Run Tests
```bash
pytest tests/ -v --cov=. --cov-report=term-missing
```

## ✅ Validation Pipeline

The `PharmaceuticalInventoryOrchestrator` runs all data through a 7-stage pipeline before and after agent execution:

1. **Input Validation** — Schema checks and type enforcement (Pydantic v2)
2. **Range Validation** — Business-rule bounds (quantities, dates, thresholds)
3. **Referential Integrity** — FK consistency across datasets
4. **Cross-Validation** — Inter-dataset logical consistency
5. **Output Validation** — Agent recommendation sanity checks
6. **Quality Scoring** — Completeness, accuracy, consistency metrics
7. **Quality Report** — Human-readable report with pass/fail summary
